手机使用和部署方式

这是一个纯前端静态项目，部署后不需要服务器，适合直接放到 GitHub Pages。

1. 手机打开即可用

页面记录默认保存在当前浏览器本地，离线也能继续补录检查和药量。

2. 可加到主屏幕

iPhone 可用 Safari 的“添加到主屏幕”；Android 可在浏览器菜单里安装，体验会更像 App。

3. 云端同步更适合 JSON

推荐把文字记录同步到 GitHub，化验单图片仍以本机保存和手动导出备份为主，更安全也更稳定。

复查与调药时间线

病人基础信息

保留手术时间、病理信息和自己的提醒，主要用来辅助看历史记录。

GitHub 云端同步

适合部署在 GitHub Pages 上后手动同步 JSON 记录。图片仍建议只作本地辅助，不建议一起长期同步，也不要在公共设备保存 Token。

拍化验单并补记录

建议先拍照，再照着把常用数值补进来。重点还是把历史复查记录补完整。

化验单图片

先上传化验单，再在旁边对照录入；保存后图片会绑定到这次复查记录。

文字辅助解析
